# Candidate Workflow Set Checklist

- The candidate request brief names the trigger, sponsor, desired terminal outcome, and downstream handoff need.
- The selection criteria define direct fit, composition pressure, adaptation pressure, material gaps, and evidence expectations.
- The workflow candidate matrix compares at least three candidates when the portfolio size permits.
- The builder baseline is explicit when it exists in the workflow capability snapshot.
- The workflow gap analysis explains whether the portfolio posture is `direct_fit`, `compose_needed`, `adapt_needed`, or `material_gap`.
- The candidate-route-posture artifact names the ranked candidates and what the downstream strategy layer should decide next.
- The human-facing candidate-workflow-set package is consistent with the machine-readable summary.
- `candidate_workflow_set_summary.json` includes comparison candidates, ranked candidates, recommended candidate workflows, builder baseline, posture, authoritative artifacts, next action, and `ready_for_strategy_selection`.
- The package stops at candidate-set publication and does not auto-run or auto-select downstream workflows.
