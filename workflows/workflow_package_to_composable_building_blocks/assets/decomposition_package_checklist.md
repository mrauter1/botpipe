# Decomposition Package Checklist

- Capture the selected workflow decomposition surface and preserve the baseline parent workflow surface.
- Record authoritative evidence in `decomposition_evidence_manifest.json`.
- Frame the decomposition request and acceptance boundary explicitly.
- Publish `extraction_strategy.md`, `building_block_interface_contracts.json`, `parent_rewrite_plan.md`, and `regression_guardrails.md`.
- Build `candidate_decomposition_surface/`, `candidate_building_block_index.json`, `decomposition_build_report.md`, and `candidate_diff_summary.md`.
- Derive `candidate_decomposition_manifest.json` deterministically from the candidate surface and building-block index.
- Publish `decomposition_verification_report.md`, `composition_migration_guide.md`, `promotion_record.md`, `rollback_plan.md`, and `workflow_decomposition_receipt.json`.
- Keep the authoritative selected workflow package unchanged before explicit promotion.
