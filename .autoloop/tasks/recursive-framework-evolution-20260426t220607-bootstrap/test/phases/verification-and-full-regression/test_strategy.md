- Preserved invariants checked:
  - The optimizer remains candidate-only, does not rerun the selected workflow, and does not execute ablations even when `optimization_depth="ablation"`.
  - Selected-workflow source immutability is checked both on the publish failure path and on a successful end-to-end run.
  - `run_statuses` remains run-level selection while `route_tags` filters step-level published evidence only.
  - Refinement accepts optimization evidence additively without auto-materializing adversarial cases or treating unablated candidates as proof.
- Edge cases:
  - Explicit `run_refs` selection.
  - Eligible runs with no matching `route_tags`.
  - Historical runs missing Plan-1 observability.
  - `no_eligible_trace_evidence` and `insufficient_evidence` short-circuit publication paths.
- Failure paths:
  - Invalid `run_ref` shapes and missing observability files.
  - Selected-workflow source drift before package publication.
  - Malformed candidate artifacts and scorecard/candidate-count contradictions.
  - Unsupported optimization evidence kinds or wrong target workflow during refinement handoff.
- Flake risk and stabilization:
  - Tests are filesystem-local, use seeded run directories plus fake-provider scripts, and avoid network, clock, and nondeterministic ordering dependencies.
  - Scoped and full-suite validation runs use `./.venv/bin/pytest` because `pytest` is not available on PATH in this repo environment.
- Known gaps:
  - Repeated Pydantic warnings from optimizer contract models that expose a `schema` field remain warning-only and are not normalized by this phase.
