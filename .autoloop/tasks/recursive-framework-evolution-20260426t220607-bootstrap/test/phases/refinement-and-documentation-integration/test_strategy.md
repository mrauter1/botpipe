# Test Strategy

- Task ID: recursive-framework-evolution-20260426t220607-bootstrap
- Pair: test
- Phase ID: refinement-and-documentation-integration
- Phase Directory Key: refinement-and-documentation-integration
- Phase Title: Refinement And Documentation Integration
- Scope: phase-local producer artifact
- Behaviors covered:
  - Refinement accepts supported optimization evidence entries through `refinement_evidence_path` and normalizes them into workflow-local baseline refinement evidence artifacts.
  - Refinement treats optimization candidates as candidate-only and unproven without stronger ablation or rerun evidence.
  - Refinement does not auto-materialize adversarial cases into eval-suite execution artifacts.
  - Refinement rejects invalid optimization evidence inputs when `target_workflow_id` drifts or an unsupported evidence `kind` is supplied.
  - Baseline doc assertions freeze the optimizer boundary: trace-consuming, candidate-only, non-mutating, non-default-ablation, with `run_refs` identity and `run_statuses`/`route_tags` separation.
- Preserved invariants checked:
  - Existing refinement workflow topology and publish-time manifest validation remain intact.
  - The optimization handoff stays additive; baseline evaluation evidence is still required.
  - `optimization_ablation_results` is stronger evidence than candidate estimates, but candidates alone do not count as proof.
- Edge cases:
  - No optional optimization evidence path supplied.
  - Optional evidence includes adversarial cases that must remain non-materialized.
- Failure paths:
  - Unsupported optimization evidence kinds are rejected before normalization.
  - Optimization evidence for the wrong selected workflow is rejected before refinement proceeds.
- Flake risk and stabilization:
  - Tests are filesystem-local and use the fake scripted provider; no network, time, or ordering dependency is introduced.
- Known gaps:
  - This phase does not broaden into optimizer workflow runtime tests; those remain covered in the optimizer phase test files.
