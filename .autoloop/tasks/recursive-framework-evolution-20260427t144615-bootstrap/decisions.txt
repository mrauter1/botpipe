Run step-level `before` hooks exactly once per step attempt boundary before artifact resolution and required-input checks, so hooks may materialize inputs or mutate state without interacting with provider retry loops.
Normalize `after` hook results before final route-specific artifact enforcement and treat hook-driven route changes as workflow-authored control decisions, so final-route missing artifacts are no longer marked provider-attributable or retriable after a hook override.
Keep simple `workflow_step(...)` lowering on the existing generated child-workflow handler path for compatibility, but compile those lowered nodes as `kind="workflow"` so the engine can honor workflow-step-specific hook ordering and loop topology without adding a second child-workflow execution engine.
Until workflow steps have an explicit handoff-delivery path into child invocations, treat them like system steps for route-handoff scheduling and drop queued handoffs targeted at `kind="workflow"` nodes instead of persisting undeliverable `PendingHandoff` entries.
