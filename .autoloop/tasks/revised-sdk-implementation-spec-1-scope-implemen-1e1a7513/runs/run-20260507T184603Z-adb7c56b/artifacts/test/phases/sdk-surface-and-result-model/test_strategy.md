- Scope: phase-local producer artifact
- Behaviors covered:
  - Root-package export surface includes the requested concrete step classes and retention/result dataclasses via `tests/unit/test_simple_surface.py`.
  - SDK workflow results expose `ResultArtifact` objects with preserved `kind`, `schema`, `required`, `qualified_name`, `source_path`, and `promoted` metadata via `tests/unit/test_sdk_facade.py`.
  - `ResultArtifact` happy path covers `read_json()`, `read_model()`, and `materialize()`.
  - `StepResult` exposes `status` and keeps `value=None` on step results.
- Preserved invariants checked:
  - Runtime workflow execution still returns an `ArtifactMap` at the SDK boundary.
  - Public SDK artifacts remain readable after the public-surface swap from `ArtifactHandle` to `ResultArtifact`.
  - `client.step(...)` still preserves route/status reporting while no longer surfacing `workflow_result.output` as `StepResult.value`.
- Edge cases:
  - Explicit `None` message handling remains covered.
  - `ResultArtifact.read_model()` failure when no schema is declared.
- Failure paths:
  - `ResultArtifact.read_model()` rejects non-Pydantic/dict schemas for model parsing.
  - `StepResult.value` regression guard uses a stubbed `WorkflowResult` with a non-`None` output payload to ensure the test fails if the old mirroring behavior returns.
- Known gaps:
  - Full runtime execution of these tests could not be performed in this shell because `pytest` and `pydantic` are unavailable here.
  - Retention plumbing, sentinel handling, and cleanup behaviors remain out of scope for this phase.
