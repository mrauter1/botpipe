Session bindings become explicit-only: the engine must look up declared slots and fail clearly when a step's session was never opened; it may not auto-open, infer, or derive bindings from workflow state.
`autoloop_v3.runtime` must stop owning phases, pair/phase CLI flags, phase-plan parsing, Autoloop artifact conventions, and slot-name-specific session paths; those move to `autoloop_v1`-owned helpers if still required for parity.
If the repo-root `workflow/` package remains, it is only a strict re-export of the canonical surface and must not behave as a compatibility shim.

The strict core now recognizes only `on_outcome` as global middleware; `on_verdict` is no longer a compatibility alias and is treated as either a normal step handler name or an orphan-handler validation error.
Session opening is fully explicit in the strict core: `SessionLifecycle` is removed, the engine never auto-opens missing step sessions, and the resulting runtime error names both the required slot and the step that tried to use it.
The repo-root `workflow` package and `runtime.loader` are now plain strict pass-through surfaces; they do not map to a legacy base class and they do not inject authoring symbols into workflow modules before import.

The generic runtime workspace now owns only task/run roots, request snapshots, events, checkpoints, and session directories; phase plans, phase selection files, raw logs, and decisions ledgers are no longer runtime-core responsibilities.
Filesystem session persistence now defaults to `sessions/{slot}.json` for unscoped bindings and `sessions/scopes/{scope_key}/{slot}.json` for scoped bindings; if Autoloop-v1 parity needs legacy filenames later, it should supply a workflow-owned path resolver instead of reintroducing slot-name branching in the runtime.
Generic runner/CLI config is reduced to workflow-agnostic controls (`intent_mode`, `max_steps`); prompt resolution is anchored on the loaded workflow file path plus repo/cwd roots, so workflows must carry explicit prompt paths instead of relying on runtime-specific search-root hacks.
