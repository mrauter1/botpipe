- `IMP-001` `blocking` [`core/engine.py:1097`, `core/engine.py:1598`, `core/engine.py:1717`]: `after_producer` hooks cannot return `RequestInput`, `Goto`, or `Fail`. The accepted hook contract says hooks may return runtime controls, and `produce_verify_step` exposes `after_producer` as a public hook surface. Today the producer path calls `_run_after_hook(... candidate_event=None ...)`, which forces `allow_direct_control=False` and raises `WorkflowExecutionError` instead of executing the control. Concrete failure: a workflow that wants to pause for human input or short-circuit to another step after producer output but before verifier dispatch will crash rather than suspend/jump/fail directly. Minimal fix: propagate `_DirectRuntimeControl` through the candidate-free after-hook path as a first-class result, then handle it centrally in the pair-step execution/finalization flow instead of gating direct controls on `candidate_event is not None`.

- `IMP-002` `blocking` [`core/primitives.py:90`, `core/engine.py:1866`, `core/engine.py:457`, `core/engine.py:541`]: `Goto.handoff` is accepted by the public API but dropped entirely, and direct `Goto` transitions are not checkpointed before execution. `_normalize_direct_runtime_control()` resolves the target and emits a runtime event, but it never carries `control.handoff` into pending handoff scheduling, so the next provider-mediated step silently loses the author-specified handoff message. Separately, the run loop checkpoints only terminal `AWAIT_INPUT`/`FAIL` cases; `Goto` falls through to `current_step_name = destination` with no checkpoint of the validated runtime control, which contradicts the requested “validate, checkpoint, execute” semantics and weakens resume/crash fidelity for direct controls. Minimal fix: extend `_DirectRuntimeControl` to carry handoff metadata, route it through the existing pending-handoff scheduler, and persist a checkpoint for direct non-terminal controls before advancing to the target step, ideally in the same central transition path that already handles terminal direct controls.
