Treat `autoloop.policy.Policy` as the only public policy facade; keep `ProviderPolicyOverride` accepted internally for compatibility but remove any public export, constructor, example, or test expectation named `PolicyOverride`.
Make `autoloop/runtime/provider_policy_resolver.py` the single merge-order owner for runtime config default policy, SDK client default policy, workflow policy, SDK run policy, step policy, and inline explicit operation policy; SDK methods should pass layers through `RunnerOptions` rather than pre-merging them.
The spec requires intentional public API breaks: reject SDK `root=` and `typed_input=` compatibility, do not add `parameters=`, and reject raw strings for enum-backed public `Policy(...)` fields.
