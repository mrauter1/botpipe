<autoloop-decisions-header version="1" block_seq="12" owner="implementer" phase_id="scoped-state-worklists" pair="implement" turn_seq="1" run_id="run-20260502T022441Z-8f34cad8" ts="2026-05-02T12:45:34.305207+00:00" />
Scoped item runtime state is now composed through `Worklist.runtime_item_state_model`: built-in `status`, `last_step`, and `last_route` are always present for active scoped items and declared `item_state=` fields may not shadow those reserved names.
Worklist helper methods mutate selection/status only, synchronize the bound context immediately, emit `worklist_refreshed` / `worklist_status_set` / `worklist_status_reset` / `worklist_advanced` / `worklist_exhausted` runtime events from the current execution-source metadata, and rely on normal checkpointing rather than implicit rollback or routing.
Route effects are removed with no compatibility shim after helper parity: route advancement/status workflows now use `on_taken` plus `ctx.current_worklist` helpers, and route finalization re-resolves artifacts after `after` hooks but before `on_taken` so validation still targets the finalized item that produced the route.
Invalid hook-selected direct controls are checkpointed as `runtime_control_validation` failures with the current mutated state preserved, instead of being collapsed into a generic hook failure.
Worklist cache entries must remain validated full-source snapshots: helper status mutations may merge selected-item updates into the cached full snapshot for immutable sources, but `refresh()` and helper validation must bypass the cache and force a source reload so mutable/artifact-backed worklists cannot go stale.
